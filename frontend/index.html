📋 How to Submit Your Fix

Debugging is done entirely on your own machine. The server never runs your local code — you must push your fixes to GitHub and validate using the ⚡ Validate tab.
Step 1 — Download

Download the section ZIP from the Dashboard. Extract and open the files in your editor.

⬇
Step 2 — Fix the Bugs

Read the README.md inside the section folder. Find & fix the intentional bugs in the source files.

🐛
Step 3 — Run Tests Locally

Run pytest sectionX-*/tests/ -v in your terminal to verify your fixes pass before pushing.

✅
Step 4 — Push & Validate

Push to your own GitHub repo, then go to ⚡ Validate and paste the URL to get your score.

🚀
⚠ Important Rules
  • You may only validate a repo that belongs to your own GitHub username.
  • Multiple submissions are allowed — only your highest score per section is kept.
  • Sections 2, 3 and 4 are locked until you pass Section 1.
  • Do not share your fixes with other participants during the event.
  • Each section has a timer deadline — validate before time runs out.
